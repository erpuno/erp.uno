N2O ECO

SYNOPSIS

N2O ECO defines a full stack specifation along with reference implementations for functional languages. This language agnostic stack is a story of functional programming and communication technologies manifested as a series of minimalistic applications and extensions to base libraries of Standard ML, Haskell, and Erlang.

O1 ECO

O1 ECO is a Standard ML N2O implementation. Standard ML is a certified language with formally defined operational semantics. Here is yet simple CML loop with INetSock.TCP spawned acceptors. Simple enough to handle N2O protocols inside the TCP server.

  • o1/n2o — N2O 0.11 ML

O3 ECO

O3 ECO introduces N2O protocols for Haskell programming language. The aim is to be compatible at transport level with Erlang version of N2O. Here is full implementation of Nitrogen Web Framework in Haskell along with simple yet powerful N2O core and HTML5 example driven by static and WebSocket servers.

  • o3/n2o — N2O 0.11 HS
  • o3/nitro — NITRO 0.11
  • o3/sample — SAMPLE 0.11

O7 ECO

O7 ECO defines ecosystem, a full-stack specification and implementation for banking, messaging and platform development. It specifies the way to scale applications, protocols, processes, services, and storage. Ready for high-speed, low-latency MQTT, TCP, QUIC, UDP, and WebSocket applications.
