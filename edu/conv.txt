Алгоритми конвертації шкал оцінювання
ERP/1: Освіта (довідник «Типи оцінок»)
1. Загальні принципи конвертації

Конвертація відбувається автоматично при розрахунку підсумкової оцінки, формуванні звітності та синхронізації з ЄДЕБО.
Основна мета — прозорість і відповідність державним нормам.
Всі алгоритми зберігаються в довіднику GradeScaleConversion або як rules у BPE-процесах.
Конвертація фіксується в історії журналу (audit).

2. Основні алгоритми
2.1. 12-бальна → ECTS (європейська шкала)
erlangconvert_12_to_ects(Grade) when Grade >= 10 -> 'A';
convert_12_to_ects(Grade) when Grade >= 9  -> 'B';
convert_12_to_ects(Grade) when Grade >= 8  -> 'C';
convert_12_to_ects(Grade) when Grade >= 7  -> 'D';
convert_12_to_ects(Grade) when Grade >= 6  -> 'E';
convert_12_to_ects(Grade) when Grade >= 4  -> 'FX';
convert_12_to_ects(_)       -> 'F'.
2.2. 12-бальна → 100-бальна (процентна)
erlangconvert_12_to_percent(Grade) ->
    round((Grade / 12) * 100).
2.3. 100-бальна → 12-бальна
erlangconvert_percent_to_12(Percent) when Percent >= 95 -> 12;
convert_percent_to_12(Percent) when Percent >= 86 -> 11;
convert_percent_to_12(Percent) when Percent >= 78 -> 10;
convert_percent_to_12(Percent) when Percent >= 71 -> 9;
convert_percent_to_12(Percent) when Percent >= 64 -> 8;
convert_percent_to_12(Percent) when Percent >= 57 -> 7;
convert_percent_to_12(Percent) when Percent >= 50 -> 6;
convert_percent_to_12(Percent) when Percent >= 43 -> 5;
convert_percent_to_12(Percent) when Percent >= 36 -> 4;
convert_percent_to_12(Percent) when Percent >= 29 -> 3;
convert_percent_to_12(Percent) when Percent >= 22 -> 2;
convert_percent_to_12(_)       -> 1.
2.4. Компетенційна шкала → Числова

Розвинена → 10–12
Базова → 7–9
Початкова → 4–6
Не сформована → 1–3

2.5. Загальна формула підсумкової оцінки за семестр
erlangfinal_grade(CurrentWeights, FinalWeights) ->
    CurrentAvg = weighted_average(CurrentWeights),
    FinalAvg   = weighted_average(FinalWeights),
    round(CurrentAvg * 0.6 + FinalAvg * 0.4).

4. Реалізація в системі

BPE: Виклик конвертації як serviceTask у процесі оцінювання.
API: Ендпоінт /api/grade/convert з параметрами source, value, target.
Журнал: Зберігається як оригінальна оцінка + конвертовані значення.
Звітність: Автоматична конверсія в потрібний формат для МОН / ЄДЕБО.
Валідація: Перевірка порогових значень перед збереженням.

5. Приклад використання в BPE (serviceTask)
erlangaction({serviceTask, 'ConvertGrades'}, Proc) ->
    Converted = convert_grades(Proc#process.docs),
    {reply, Proc#process{docs = Converted}}.
Ці алгоритми забезпечують єдину точку правди при роботі з різними шкалами.
