# ERP.uno Quick Start Deployment Guide

## Prerequisites

Your environment has:
- ✓ kubectl CLI
- ✓ Helm 3
- ✓ Kubernetes cluster (1 node, ~7GB RAM)
- ✓ CoreDNS available
- ✓ StorageClass: standard
- ⚠ Cluster is tight (recommended 16GB, you have 7GB)
- ⚠ No Ingress controller yet
- ⚠ Image registry: registry.erp.uno (ensure accessible)

# Step 1: Initialize namespace

## Deployment Options

### Option 1: Quick Deploy (Recommended)

```bash
bash /Users/tonpa/depot/erpuno/erp.uno/cd/deploy-start.sh
```

This will:
1. Verify Kubernetes cluster
2. Create namespace
3. Set up storage
4. Deploy Helm chart with 25 services
5. Wait for critical services (ns-dns, kvs-database, ias-auth)
6. Show summary + next steps

**Estimated time**: 3–5 minutes

---

### Option 2: Manual Helm Deploy

```bash
# Clone repo (already done at /Users/tonpa/depot/erpuno/erp.uno/cd/erp.uno)
cd /Users/tonpa/depot/erpuno/erp.uno/cd/erp.uno/cd/helm

# Install chart
helm install erp-uno . \
  --namespace erp-uno \
  --create-namespace \
  --values values.yaml

# Monitor deployment
kubectl get pods -n erp-uno -w

# Wait for readiness
kubectl wait --for=condition=ready pod \
  -l service=kvs-database \
  -n erp-uno \
  --timeout=300s
```

---

### Option 3: ArgoCD (GitOps)

```bash
# Install ArgoCD (if not already installed)
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Apply ERP.uno ArgoCD Application
kubectl apply -f /Users/tonpa/depot/erpuno/erp.uno/cd/erp.uno/cd/argocd/application.yaml

# Monitor sync
kubectl get application -n argocd -w
```

ArgoCD will auto-sync on Git push to `https://github.com/erpuno/erp.uno`.

## Local Container Registry (macOS + Kubernetes)

### Quick Setup (Automated)

```bash
# Step 1: Initialize
bash /Users/tonpa/depot/erpuno/erp.uno/cd/init.sh

# Step 2: Deploy Docker Registry to K8s
bash /Users/tonpa/depot/erpuno/erp.uno/cd/setup-registry.sh

# Step 3: (Optional) Deploy Registry UI
bash /Users/tonpa/depot/erpuno/erp.uno/cd/setup-registry-ui.sh
```

This will:
1. Create `erp-uno` namespace
2. Deploy Docker Registry (50Gi storage)
3. Add `registry.erp.uno` to `/etc/hosts`
4. Configure Docker daemon (`insecure-registries`)
5. Set up port-forward (localhost:5000)
6. Test connectivity

**Estimated time**: 2–3 minutes

---

### Build and Push Images to Local Registry

```bash
# Build Docker image
docker build -t registry.erp.uno:5000/erpuno/myservice:2024.6.15 .

# Push to local registry
docker push registry.erp.uno:5000/erpuno/myservice:2024.6.15

# Verify push succeeded
curl http://registry.erp.uno:5000/v2/erpuno/myservice/tags/list
```

---

### Browse Registry Images

#### Option A: Registry UI (Web)

After deploying Registry UI:

```bash
# Terminal 1: Start port-forward
kubectl port-forward -n erp-uno svc/registry-ui 8080:8080

# Terminal 2: Open browser
# http://localhost:8080
```

You'll see:
- All repositories (namespaces)
- All images in each repo
- All tags for each image
- Image size, digest, creation date
- Ability to delete images

#### Option B: Command Line

```bash
# List all repositories
curl http://localhost:5000/v2/_catalog

# List tags for specific image
curl http://localhost:5000/v2/erpuno/myservice/tags/list

# Example output:
# {"name":"erpuno/myservice","tags":["2024.6.15","latest"]}

# Get image digest
curl -I http://localhost:5000/v2/erpuno/myservice/manifests/2024.6.15
```

### Update Helm Values to Use Local Registry

Edit `/Users/tonpa/depot/erpuno/erp.uno/cd/erp.uno/cd/helm/values.yaml`:

```yaml
global:
  registry: registry.erp.uno:5000  # Changed from registry.erp.uno
  imagePullPolicy: IfNotPresent
```

Or deploy with override:

```bash
helm upgrade erp-uno . -n erp-uno \
  --set global.registry=registry.erp.uno:5000
```

## Troubleshooting

### Pods stuck in Pending
```bash
kubectl describe pod -n erp-uno <pod-name>
# Check: PVC binding, node resources, storage availability
```

### Pod CrashLoopBackOff
```bash
kubectl logs -n erp-uno <pod-name>
# Check: readiness probe timeout, application startup errors
```

### Service not responding
```bash
# Check endpoint availability
kubectl get endpoints -n erp-uno

# Check service DNS from inside pod
kubectl exec -it -n erp-uno <pod-name> -- nslookup <service>.erp-uno.svc.cluster.local
```

### Scale down for resource-constrained cluster

Your cluster is tight (~7GB). To reduce footprint:

```bash
# Edit values.yaml and reduce replicas
kubectl edit cm -n erp-uno erp-uno-config

# Or delete non-critical services
helm upgrade erp-uno /Users/tonpa/depot/erpuno/erp.uno/cd/erp.uno/cd/helm \
  -n erp-uno \
  --set services.ai-generation.enabled=false \
  --set services.olap-analytics.enabled=false \
  --set services.prometheus.enabled=false
```
